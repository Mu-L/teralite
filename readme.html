hello, thanks for downloading

(un)frequently asked questions

.pck not found at path '.' error

teralite.exe has to be in the same folder as the teralite.pck or it won't work. if you want to have access to the game somewhere else, just make a shortcut of teralite.exe

will get game get updated?

maybe. i started on this game in august 2020, which makes this the longest time ever spent on a project, nothing else even comes close. so i'm a bit ready to move on. however, i will definitely be doing some bug fix updates, and i also had some ideas planned such as:

  • keybinds and controller remaping (also better button prompts)
  • online leaderboards for level times
  • a endless roguelike tower mode
  • porting to godot 4 (preformance boost), and improving the code (particularly AI it's horrible)
  • goblins

but like i said i kind of... never want to touch this game and it's horrible AI code again. so i don't really feel comfortable promising any of these things (especially if the game gets like 5 downloads or something) and if they do happen it will be a while from now. for the immediate future i'll be putting this project on the backburner, in the basement, locked away

preformance apology

the game can be suprisingly intensive. it's better than yandare simulator (i hope?) but it's still way worse than any 2d pixel art game ever should be, to the point where I have been seriously considering using a custom engine for my future games. i think it's due to a combination of my unoptimized code, as well as godot's lighting (overlapping lights cause a huge dip). if you're having issues, turning off lighting can help a lot. i might fix it eventually, but like i said in the section above, i'm sort of done with teralite. sorry.

particles aren't working

if you're using a really old CPU with integrated graphics that doesn't support OpenGL ES 3, the game will fallback to OpenGL ES 2. this mostly works okay, but the game will run worse and particle effects won't work. sorry! i might make a "CPU particles" option in the future, but it would be a hassle because i would have to make a 2nd version of every particle effect
